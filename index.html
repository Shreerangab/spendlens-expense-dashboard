About This Dashboard

How Currency Conversion Works

All expenses are converted to USD using a static snapshot of exchange rates dated 2026-05-01. The conversion formula is straightforward: USD = original amount ÷ rate, where each rate represents how many units of that currency equal one US dollar. For example, if 1 USD = 83.47 INR, an expense of ₹8,200 becomes $98.24 USD.

Why a static rate file?

Using a single, versioned rate file means every report is reproducible — running the same data next month gives the same output. This is critical for board reporting where figures must be auditable and consistent. Live API rates would cause the same transaction to show different USD values depending on when the report is generated.

Adding a 25th currency

The rates object is a simple key-value map. Adding a new currency requires one line: the ISO code and its USD rate. No logic changes are needed — the conversion function reads from the map dynamically. The add-expense form and conversion pipeline will pick it up automatically.

Handling null or missing rates

If a rate entry is null or undefined, dividing by it would produce Infinity or NaN — both of which would silently corrupt totals. The conversion function guards against this by checking for a valid, positive rate before dividing. Expenses with missing rates are flagged as "rate unavailable" rather than silently treated as zero or skipped.
